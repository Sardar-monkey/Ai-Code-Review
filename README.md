# AI Code Review CLI

Минималистичный инструмент для ревью кода на базе ИИ, который анализирует файлы через **DeepSeek API** и выдаёт структурированную обратную связь — прямо из терминала.

Создан в учебных целях, чтобы понять, как работают под капотом такие инструменты, как **Cline**, **Aider**, **OpenCode** и **OpenHands**.

## Возможности

- ✅ **Ревью одного файла** — `python review.py app.js`
- ✅ **Ревью директории** — рекурсивно проверяет все поддерживаемые файлы в папке
- ✅ **Выбор модели** — `--model deepseek-reasoner`
- ✅ **Режимы фокуса** — `--focus security`, `--focus performance`, `--focus readability`
- ✅ **Режим Diff** — `--diff old.py new.py` для ревью изменений между двумя версиями
- ✅ **Экспорт в Markdown** — `--output review.md` сохраняет результат в файл
- ✅ **Цветной вывод в терминале** — ANSI-цвета без внешних библиотек
- ✅ **Обработка ошибок** — без Python traceback'ов, только понятные сообщения
- ✅ **API-ключ не в коде** — загружается из файла `.env` или переменной окружения

## Поддерживаемые типы файлов

## Установка

### 1. Клонируйте репозиторий

```bash
git clone <url-репозитория>
cd Ai-Code-Review
```

### 2. Настройте API-ключ

Отредактируйте `.env` и вставьте ключ:

```
DEEPSEEK_API_KEY=sk-ваш-настоящий-ключ
```

> **Получить ключ:** [https://platform.deepseek.com/api_keys](https://platform.deepseek.com/api_keys)

Либо установите переменную окружения:

```bash
# Linux/macOS
export DEEPSEEK_API_KEY=sk-ваш-ключ

# Windows CMD
set DEEPSEEK_API_KEY=sk-ваш-ключ

# PowerShell
$env:DEEPSEEK_API_KEY="sk-ваш-ключ"
```

### 3. Зависимости не требуются!

Инструмент использует только **стандартную библиотеку** Python — `pip install` не нужен.

## Использование

### Базовое ревью

```bash
python review.py samples/app.js
```

### Ревью с фокусом

```bash
python review.py samples/app.py --focus security
```

### Использовать другую модель

```bash
python review.py samples/app.js --model deepseek-reasoner
```

### Сохранить ревью в Markdown

```bash
python review.py samples/app.py --output review.md
```

### Проверить всю директорию

```bash
python review.py samples/ --focus readability
```

### Режим Diff (сравнение двух файлов)

```bash
python review.py --diff samples/app_old.py samples/app.py
```

### Комбинировать опции

```bash
python review.py samples/ --model deepseek-reasoner --focus security --output review.md
```

## Обработка ошибок

| Сценарий                    | Сообщение                                                     |
|-----------------------------|---------------------------------------------------------------|
| Файл не найден              | `✖ Ошибка: Файл не найден: nonexistent.js`                   |
| Неподдерживаемое расширение | `✖ Ошибка: Неподдерживаемое расширение '.txt'. Поддерживаются: .js, .py, .html, .css` |
| Нет API-ключа               | Показывает инструкцию по созданию `.env` или установке переменной окружения |
| Сетевая ошибка              | `✖ Ошибка: Сетевая ошибка: ...`                              |
| Ошибка API                  | `✖ Ошибка: DeepSeek API вернул HTTP 401: ...`                |

## Как это работает

1. **Разбор аргументов** — ручной разбор через `sys.argv` (без argparse/click/typer)
2. **Чтение файла** — чтение содержимого файла в кодировке UTF-8
3. **Формирование запроса** — создаётся системный промт + сообщение пользователя с кодом
4. **HTTP-запрос** — отправляется POST-запрос на `https://api.deepseek.com/v1/chat/completions` с:
   - **URL**: `https://api.deepseek.com/v1/chat/completions`
   - **Метод**: `POST`
   - **Заголовки**: `Content-Type: application/json`, `Authorization: Bearer <ключ>`
   - **Тело**: JSON с полями `model`, `messages`, `temperature`, `max_tokens`
5. **Разбор ответа** — извлекается текст ревью из ответа API
6. **Вывод** — печать с ANSI-цветами или сохранение в Markdown

## Замечания по безопасности

- **API-ключ НИКОГДА не зашит** в исходном коде
- Файл `.env` указан в `.gitignore` — он не попадёт в репозиторий
- Если вы случайно закоммитите ключ в публичный репозиторий, **боты найдут его в течение нескольких минут** и израсходуют ваши API-кредиты
- Всегда используйте `.env.example` как шаблон (без реальных ключей)

## Сравнение: этот инструмент vs Cline

| Возможность              | Этот инструмент | Cline |
|--------------------------|-----------------|-------|
| Ревью кода               | ✅ Да           | ✅ Да |
| Редактирование файлов    | ❌ Нет          | ✅ Да |
| Запуск команд в терминале| ❌ Нет          | ✅ Да |
| Создание файлов          | ❌ Нет          | ✅ Да |
| Многошаговый диалог      | ❌ Нет          | ✅ Да |
| Интеграция с VS Code     | ❌ Нет          | ✅ Да |
| Интеграция с Git         | ❌ Нет          | ✅ Да |
| Поиск в интернете        | ❌ Нет          | ✅ Да |
| Поддержка MCP-серверов   | ❌ Нет          | ✅ Да |

Оба инструмента делают одно и то же по сути: **отправляют код + промт в LLM и показывают ответ**. Разница в том, что Cline — это полноценный агент, который может действовать на основе ответа (редактировать файлы, запускать команды), а этот инструмент — узкоспециализированная утилита для ревью кода.
